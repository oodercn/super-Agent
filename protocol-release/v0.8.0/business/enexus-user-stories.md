# 企业版 Nexus (eNexus) 用户故事集

## 1. 概述

### 1.1 文档目的
本文档基于《企业版 Nexus 业务驱动设计》，从企业运营本质出发，整理用户故事，为后续需求分析和功能设计提供基础。

### 1.2 核心角色定义

#### 企业管理者
- **职责**：制定战略、分配资源、监督执行
- **关注点**：企业盈利、风险控制、决策支持、团队管理、信息透明

#### 员工（人 + Nexus 客户端）
- **职责**：执行任务、反馈信息、提出建议
- **关注点**：工作效率、任务完成质量、个人成长、团队协作

#### LLM（企业大脑）
- **职责**：分析数据、提供洞察、辅助决策
- **关注点**：数据质量、模型准确性、响应速度、知识更新

### 5.2 核心需求总结

1. **北向协议框架需求**：
   - 实现 sceneEngine 能力
   - 为 Nexus 客户端用户提供数据能力 API
   - 确保客户端向服务端的调用实现

2. **场景能力自循环依赖原则需求**：
   - 服务端自身的场景能力管理
   - 能力的合理划分和支撑架构
   - 生产效率监控、质量控制、成本控制等场景

3. **用户角色区分需求**：
   - Nexus 客户端用户：提供数据能力 API
   - Web 页面用户：仅限于系统办公引入，个人设备管理、能力管理等模块从 Web 端移除

4. **LLM 介入需求**：
   - 主动介入：异常检测、趋势预测、优化建议
   - 被动响应：问答响应、任务执行响应、决策支持响应
   - 协作介入：人机协作决策、人机协作执行、人机协作学习
