# Ooder AI Bridge 协议主文档 - v0.8.0

## 1. 协议概述

### 1.1 协议背景

v0.8.0 版本是 Ooder Agent 协议的企业级升级版本，核心聚焦于**北向协议**的全面设计，为 LLM 企业AI大脑提供标准化服务接口，实现 AI 如何获取信息、下达指令、协调业务数据和人员协同。

### 1.2 协议目标

- **LLM 服务接口标准化**：定义 AI 大脑与企业系统的交互规范
- **NLP 语言规范**：围绕自然语言处理设计交互模型
- **管理者意志体现**：将管理者的意志转化为可执行的 AI 行为
- **数字资产治理**：设备、数据、Agent 等数字资源的统一管理
- **LLM 触达能力**：物理设备触达和虚拟资源触达的扩展机制
- **确定性数据支撑**：为 LLM 推理提供格式化数据和 API 支持

### 1.3 核心变更

| 变更项 | 说明 |
|--------|------|
| 北向协议规范 | 新增完整的北向协议技术规范，服务 LLM 企业AI大脑 |
| NLP 语言规范 | 定义意图类型、实体类型、关系类型、时空表达 |
| 意志表达模型 | 战略意志→战术意志→执行意志的转化模型 |
| 数字资产治理 | 设备资产、数据资产、Agent资产、资源资产分类管理 |
| LLM 记忆体系 | 感知记忆→工作记忆→情景记忆→语义记忆→程序记忆 |
| Agent 协同体系 | 协调层→通信层→执行层→管理层 |
| LLM-SDK 设计 | 两种方案的利弊分析（扩展方案 vs 独立SDK方案） |
| 确定性数据抽象 | 支持业务接入和未来数据底层规范 |

## 4. 核心概念

### 4.1 北向协议

北向协议是企业版 Nexus 为 LLM 企业AI大脑提供的服务接口规范，定义了 AI 如何获取信息、下达指令、协调业务数据和人员协同的标准方式。

**服务对象**：
- **主要服务对象**：LLM 企业AI大脑
- **次要服务对象**：企业管理者（通过 NLP 交互）
- **间接服务对象**：企业员工（通过 AI 协调）

### 4.2 数字资产治理

数字资产治理是企业管理设备、数据、Agent 等数字资源的虚拟业务，是 AI 智能化运营的基础。

**资产分类**：
- 设备资产：生产设备、网络设备、存储设备、终端设备、传感器设备
- 数据资产：业务数据、运营数据、财务数据、客户数据、知识数据
- Agent 资产：AI 智能体、自动化代理、监控代理、协调代理
- 资源资产：计算资源、存储资源、网络资源、服务资源

### 4.3 LLM 触达能力

LLM 的触达能力是指 AI 能够直接或间接地控制和管理企业物理设备和虚拟资源的能力。

**触达分类**：
- 物理设备触达：路由器、交换机、防火墙、摄像头、传感器、生产设备
- 虚拟资源触达：数据库、文件系统、API、消息队列、微服务、云服务

### 4.4 LLM 记忆体系

LLM 需要构建自身的记忆体系，以支持长期学习和知识积累。

**记忆层次**：
- 感知记忆：瞬时记忆，毫秒级
- 工作记忆：短期记忆，分钟级
- 情景记忆：长期记忆，天/月/年
- 语义记忆：永久记忆，知识存储
- 程序记忆：永久记忆，技能流程
