# MCP Agent 企业管理员使用手册

## 1. 角色概述

企业管理员是 MCP Agent 的高级使用角色，主要面向大型企业网络环境，提供完整的网络管理、服务管理、监控分析等企业级功能。

## 3. 页面功能说明

### 3.1 基础功能页面
- **仪表盘** (dashboard.html)
- **家庭功能相关页面** (home/)
- **小局域网功能相关页面** (lan/)

### 3.2 命令体系管理页面

#### 3.2.1 命令仪表盘 (command/command-dashboard.html)
- **功能**：展示命令执行状态和统计信息
- **主要信息**：命令执行成功率、执行时间分布、失败率分析
- **操作**：查看详细命令执行情况、快速定位问题命令

#### 3.2.2 命令发送 (command/command-send.html)
- **功能**：发送各种网络和系统命令
- **主要信息**：命令模板、目标设备、执行参数
- **操作**：创建命令、选择目标、执行命令、查看执行结果

#### 3.2.3 命令队列管理 (command/command-queue.html)
- **功能**：管理待执行的命令队列
- **主要信息**：队列状态、命令优先级、执行时间
- **操作**：调整命令优先级、暂停/恢复命令、删除命令

#### 3.2.4 网络链接管理 (command/network-links.html)
- **功能**：管理网络链接状态和配置
- **主要信息**：链接状态、带宽使用、延迟情况
- **操作**：创建链接、修改链接配置、监控链接状态

#### 3.2.5 命令统计分析 (command/command-stats.html)
- **功能**：分析命令执行的统计数据
- **主要信息**：执行趋势、成功率、失败原因分析
- **操作**：查看详细统计报表、导出分析数据

### 3.3 企业级功能页面

#### 3.3.1 企业仪表盘 (enterprise/enterprise-dashboard.html)
- **功能**：企业级综合监控面板
- **主要信息**：企业网络整体状态、关键业务系统状态、安全事件汇总
- **操作**：快速访问各企业级功能、查看详细业务系统状态

#### 3.3.2 服务管理 (system/service-management.html)
- **功能**：管理系统服务和应用
- **主要信息**：服务状态、启动时间、资源使用情况
- **操作**：启动/停止服务、修改服务配置、查看服务日志

#### 3.3.3 服务配置 (config/service-config.html)
- **功能**：配置系统服务参数
- **主要信息**：服务参数设置、配置版本管理
- **操作**：修改服务配置、保存配置、应用配置变更

#### 3.3.4 健康检查 (monitoring/health-check.html)
- **功能**：检查系统和服务健康状态
- **主要信息**：健康检查结果、异常服务列表、性能指标
- **操作**：执行健康检查、查看详细检查报告、处理异常情况

## 4. 操作指南

### 4.1 命令管理操作
1. **发送命令**：点击「命令体系管理」→「命令发送」，填写命令参数，选择目标设备，点击「执行」
2. **管理命令队列**：点击「命令体系管理」→「命令队列管理」，查看队列状态，调整命令优先级
3. **查看命令统计**：点击「命令体系管理」→「命令统计分析」，查看执行趋势和成功率

### 4.2 系统管理操作
1. **管理服务**：点击「企业级全功能应用」→「系统管理」→「服务管理」，启动/停止服务，修改服务配置
2. **配置服务参数**：点击「企业级全功能应用」→「配置中心」→「MCP配置」，修改服务参数
3. **执行健康检查**：点击「企业级全功能应用」→「监控与分析」→「健康检查」，点击「开始检查」

### 4.3 企业监控操作
1. **查看企业仪表盘**：点击「企业级全功能应用」→「企业仪表盘」，查看企业网络整体状态
2. **监控网络链接**：点击「命令体系管理」→「网络链接管理」，查看链接状态和带宽使用
3. **分析命令执行**：点击「命令体系管理」→「命令统计分析」，分析命令执行趋势和成功率

## 6. 常见问题

### 6.1 命令执行失败
- **原因**：目标设备离线、命令参数错误、网络连接问题
- **解决**：检查目标设备状态，验证命令参数，检查网络连接

### 6.2 服务启动失败
- **原因**：端口被占用、配置文件错误、依赖服务未启动
- **解决**：检查端口占用情况，验证配置文件，确保依赖服务已启动

### 6.3 健康检查异常
- **原因**：服务性能下降、资源使用过高、网络连接异常
- **解决**：查看详细检查报告，定位异常服务，调整服务配置或资源分配
