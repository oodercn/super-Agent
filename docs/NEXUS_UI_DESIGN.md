# Nexus 用户界面设计文档

## 1. 设计概述

### 1.1 设计目标

基于用户需求，重新设计 Nexus 的用户界面，以"能力"为核心，从普通用户 5W（Who、What、When、Where、Why）角度进行功能筛选和展示，实现个人用户与办公用户场景的合理分离。

### 1.2 用户角色定义

| 角色 | 描述 | 主要场景 |
|------|------|----------|
| 个人用户（私有） | 个人使用，关注个人效率和工具能力 | 个人技能管理、个人数据管理、个人偏好设置 |
| 个人用户（办公） | 企业环境中的个人，关注协同和协作 | 团队协作、任务分配、资源共享 |
| 管理员 | 高级定制和管理 | 能力检索、调配、切换、系统配置 |

### 1.3 设计原则

1. **能力为中心**：所有功能围绕"能力"展开
2. **场景分离**：个人场景与办公场景清晰分离
3. **渐进式展示**：从简单到复杂，引导用户逐步深入
4. **个性化定制**：支持用户自定义界面和功能

### 2.2 5W 角度功能筛选

#### 2.2.1 Who（谁在使用）

| 用户类型 | 关注点 | 功能展示 |
|---------|--------|----------|
| 个人用户 | 个人效率、个人数据 | 我的技能、我的数据、我的设置 |
| 团队成员 | 团队协作、任务分配 | 团队任务、协作场景、共享资源 |
| 管理员 | 系统管理、能力调配 | 能力管理、用户管理、系统配置 |

#### 2.2.2 What（做什么）

| 能力类型 | 功能描述 | 入口位置 |
|---------|----------|----------|
| 数据处理 | 数据抽取、转换、分析 | 能力中心 → 数据能力 |
| 文档处理 | 文档编辑、转换、管理 | 能力中心 → 文档能力 |
| 网络服务 | 网络监控、配置、管理 | 能力中心 → 网络能力 |
| 协作服务 | 消息、文件、场景协作 | 办公协作 → 协作能力 |
| 安全服务 | 认证、授权、审计 | 管理配置 → 安全能力 |

#### 2.2.3 When（什么时候）

| 时间场景 | 功能展示 | 界面元素 |
|---------|----------|----------|
| 日常使用 | 常用能力、快捷入口 | 首页快捷入口、收藏能力 |
| 定期任务 | 定时任务、周期执行 | 任务管理、执行计划 |
| 临时需求 | 能力搜索、快速调用 | 搜索栏、能力市场 |
| 突发事件 | 告警处理、应急响应 | 通知中心、告警管理 |

#### 2.2.4 Where（在哪里）

| 使用环境 | 功能适配 | 界面设计 |
|---------|----------|----------|
| 个人电脑 | 完整功能 | 全功能界面 |
| 移动设备 | 核心功能 | 简化界面、触摸优化 |
| 远程访问 | 安全访问 | VPN 集成、安全认证 |
| 局域网 | 本地优先 | 本地发现、快速连接 |

#### 2.2.5 Why（为什么）

| 使用目的 | 功能引导 | 界面提示 |
|---------|----------|----------|
| 提升效率 | 效率工具推荐 | 使用统计、效率报告 |
| 解决问题 | 问题诊断能力 | 问题分类、解决方案 |
| 学习成长 | 能力学习路径 | 学习进度、能力认证 |
| 团队协作 | 协作能力推荐 | 团队动态、协作统计 |
