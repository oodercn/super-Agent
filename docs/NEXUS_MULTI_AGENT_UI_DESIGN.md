# Nexus 多智能体用户界面设计文档

## 1. 设计概述

### 1.1 设计理念

以**智能体**为核心入口，将能力封装在智能体内部展现。用户视角从"能力做了什么"转变为"智能体（伙伴、应用场景）今天做了什么"。整体主题聚焦于**多智能体协作**。

### 1.2 核心概念

| 概念 | 描述 | 用户视角 |
|------|------|----------|
| 智能体 | 具备特定能力的 Agent，可以是助手、工具或场景 | 我的伙伴、我的助手 |
| 场景 | 多智能体协作的工作空间 | 我的工作场景、协作空间 |
| 能力 | 智能体具备的技能（封装在智能体内） | 智能体能做什么 |
| 对话 | 与智能体的交互方式 | 与伙伴沟通 |

### 1.3 设计原则

1. **智能体为中心**：所有功能围绕智能体展开，能力隐藏在智能体内部
2. **场景化协作**：多智能体在场景中协作完成复杂任务
3. **对话式交互**：通过对话与智能体沟通，LLM 功能嵌入主页面
4. **伙伴化命名**：使用"伙伴"、"助手"、"场景"等用户友好的术语
